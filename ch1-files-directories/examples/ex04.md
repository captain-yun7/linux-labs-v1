### ex04 : 로그 파일에서 원하는 정보만 골라내기 (grep, 정규표현식)

#### 1\. 실습용 로그 파일 준비

```bash
# [가짜 웹서버 로그 생성] - 다양한 상태 코드(200, 404, 500)와 IP 주소를 포함합니다.
cat << 'EOF' > web_access.log
192.168.1.10 - - [13/Jul/2025:10:00:00 +0900] "GET /index.html HTTP/1.1" 200 1543
192.168.1.12 - - [13/Jul/2025:10:00:01 +0900] "GET /about.html HTTP/1.1" 200 876
211.172.10.5 - - [13/Jul/2025:10:00:02 +0900] "POST /login HTTP/1.1" 404 201
192.168.1.10 - - [13/Jul/2025:10:00:03 +0900] "GET /styles.css HTTP/1.1" 200 1234
10.0.5.25 - - [13/Jul/2025:10:00:04 +0900] "GET /api/users HTTP/1.1" 500 50
EOF
```

#### 2\. 기본적인 필터링

```bash
# [단순 검색] - '404' 에러가 발생한 모든 로그를 찾습니다.
grep "404" web_access.log

# [대소문자 무시] - 'get' 이나 'GET' 이나 모두 찾습니다.
grep -i "get" web_access.log

# [결과 반전] - '200' 성공 코드를 제외한, 즉 문제가 있는 로그만 봅니다.
grep -v " 200 " web_access.log
```

#### 3\. 정규표현식을 이용한 고급 필터링

```bash
# [^ : 시작 패턴] - '192.168'로 시작하는 내부 IP의 접속 기록만 봅니다.
grep "^192.168" web_access.log

# [. : 임의의 한 글자] - '192.168.1.1' 뒤에 아무 숫자나 하나 오는 IP를 찾습니다.
grep "192.168.1.1." web_access.log

# [* : 0회 이상 반복] - 'GET'과 'HTTP' 사이에 어떤 문자든 와도 되는 로그를 찾습니다.
grep "GET.*HTTP" web_access.log

# [OR 조건] - 404 에러 또는 500 에러가 발생한 로그를 모두 봅니다.
grep -E "404|500" web_access.log
```
