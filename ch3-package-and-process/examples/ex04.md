### ex04 : 서비스 및 포그라운드/백그라운드 프로세스 관리

#### 1\. 서비스 관리 (systemd)

```bash
# [nginx 서비스 상태 확인] - Active 상태, 로그, PID 등을 보여줍니다.
sudo systemctl status nginx

# [nginx 서비스 중지]
sudo systemctl stop nginx

# [nginx 서비스 다시 시작]
sudo systemctl start nginx

# [nginx 서비스 재시작] - 실행 중일 때만 재시작합니다.
sudo systemctl restart nginx

# [부팅 시 nginx 자동 실행 설정]
sudo systemctl enable nginx

# [부팅 시 nginx 자동 실행 해제]
sudo systemctl disable nginx

# [nginx 서비스의 최근 로그 10줄 확인]
sudo journalctl -u nginx -n 10
```

#### 2\. 포그라운드/백그라운드 프로세스 제어 (jobs)

```bash
# [포그라운드 작업 실행] - 이 명령이 실행되는 동안 터미널은 다른 입력을 받지 못합니다.
sleep 30

# [백그라운드 작업 실행] - 명령어 뒤에 '&'를 붙이면 바로 프롬프트로 돌아옵니다.
sleep 300 &

# [현재 실행 중인 백그라운드 작업 목록 확인]
jobs

# [백그라운드 작업을 포그라운드로 가져오기] - 'jobs' 목록의 1번 작업을 가져옵니다.
fg %1

# (fg로 가져온 후) [포그라운드 작업을 일시 정지하고 백그라운드로 보내기]
# Ctrl + Z

# [일시 정지된 작업을 다시 백그라운드에서 실행] - 'jobs' 목록의 1번 작업을 실행합니다.
bg %1
```
