### ex01 : 디스크 확인 및 사용량 분석

#### 1. 디스크 전체 구조 확인

```bash
# [시스템에 연결된 디스크 목록 보기]
# lsblk는 블록 디바이스(디스크, 파티션) 목록을 트리 형태로 보여줍니다.
# 전체 디스크 구조를 파악할 때 가장 먼저 사용하는 명령어입니다.
lsblk

# [숨겨진 디바이스까지 모두 보기]
# '-a' 옵션을 주면 사용되지 않는 디바이스 등도 모두 표시합니다.
lsblk -a
```

#### 2. 파일 시스템 용량 확인

```bash
# [현재 마운트된 파일 시스템 용량 보기]
# df는 마운트된 파일 시스템의 전체/사용/남은 용량을 보여줍니다.
df

# [읽기 편한 단위로 용량 보기]
# '-h' (human-readable) 옵션을 주면 GB, MB 등 사람이 읽기 편한 단위로 보여줍니다.
df -h

# [파일 시스템 종류와 함께 보기]
# '-T' 옵션을 주면 ext4, xfs 등 파일 시스템의 타입을 함께 보여줍니다.
df -hT
```

#### 3. 특정 경로의 사용량 확인

```bash
# [현재 디렉토리의 총 사용량 계산하기]
# du는 특정 디렉토리나 파일의 디스크 사용량을 계산합니다.
# '-s' (summarize)는 총합만, '-h'는 읽기 편한 단위로 보여줍니다.
du -sh .

# [/var/log 디렉토리의 총 사용량 확인하기]
# 시스템 로그 디렉토리의 용량을 확인할 때 자주 사용합니다. (sudo 필요)
sudo du -sh /var/log

# [/var/log 하위 디렉토리별 용량 확인하기]
# '--max-depth=1' 옵션으로 하위 1단계까지만 용량을 각각 보여줍니다.
# 어떤 로그 파일이 용량을 많이 차지하는지 분석할 때 유용합니다.
sudo du -h --max-depth=1 /var/log

# [용량 순으로 정렬하여 보기]
# du의 결과를 sort 명령어로 넘겨 용량 순으로 정렬할 수 있습니다.
# '-n'은 숫자 순, '-r'은 역순(큰 것부터) 정렬입니다.
sudo du -h --max-depth=1 /var/log | sort -nr
```
